hexo系列教程：（二）hexo博客的目录结构与使用

  • Hexo
Blog

了解Hexo

在上一篇文章中，我们只是使用Hexo简单地搭建了一个站点，但是要想更好地使用它，还是需要多了解关于Hexo的一些东西。

目录结构

Hexo的目录结构如下，其中：

  1. _config.yml是整个Hexo站点的核心配置文件

  2. source/_drafts是草稿文件目录

  3. source/_posts是发表的博文目录

  4. themes是下载的主题目录

至于其他的目录暂时可以不用管。

.
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
|   ├── _drafts
|   └── _posts
└── themes

首先就是修改_config.yml配置文件，主要就是配置一下自己的站点信息，配置文件中的注释比较详细，如果不懂的话可以看下官方文档，在预览状态下去改的话网页上能即时看到效果，这一点很赞，即时渲染。

我的如下：

title: 鬼脚飞's Blog                    #网站标题
subtitle: 没有梦想 何必远方！            #简介
description: 没有梦想 何必远方！            #描述
author: hu1991die                        #网站作者
language: zh-CN                            #网站语言
email: 347431996@qq.com                    #邮箱
timezone: Asia/Hong_Kong                #时区

默认的主题是landscape，更多的主题可以在https://github.com/hexojs/hexo/wiki/Themes这儿下载，基本就是git clone到themes目录下然后修改_config.yml中的theme字段即可，也十分简单方便。

例如：Yilia主题使用

安装主题

$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

配置主题

修改hexo根目录下的_config.yml ——》 theme: yilia

更新主题（随意，想更新就更新）

cd themes/yilia

git pull

Yilia主题效果展示

详情见博客主页：http://hu1991die.com/

Hexo常见指令

Hexo的常用命令有以下几个：

$ hexo new [layout] <title>     #建立新文章，默认在_posts下，layout="draft"时发布的是草稿
$ hexo publish <filename>       #将_drafts下的文件放到_posts下，也就是发布草稿
$ hexo generate                 #生成静态网页
$ hexo server                   #启动预览服务器，开启-d选项时可以预览草稿
$ hexo deploy                   #发布到远程服务器，开启--generate选项可以在deploy前自动generate

Tips

$ hexo n  == hexo new
$ hexo p  == hexo publish 
$ hexo g  == hexo generate
$ hexo s  == hexo server
$ hexo d  == hexo deploy

写博文

好了，站点配置好了，该了解的也都了解了，现在我想发表一篇博文，怎么做呢？

  1. 在hexo根目录下，执行命令

    hexo　new "my_first_post"

  2. 在H:\hexo\source\_posts中打开这个文件（打开方式用“记事本”或者NotePad++即可），配置开头头部信息，如下所示是本文的头部信息，以---结尾。接下来，使用Markdown编辑文章就可以了。

############—
title: hexo系列教程：（二）hexo博客的目录结构与使用
layout: post
date: 2016-03-07 16:37:43
comments: true
categories: Blog
tags: [Hexo]
keywords: Hexo, Blog
description: 生命在于折腾，又把博客折腾到Hexo了。给Hexo点赞。

############—

发布博文

由于生成的都是静态页面，可以和Octopress一样直接发布在gihub上。

  1. 使用命令hexo s，本地浏览器中访问http://localhost:4000预览效果。（退出server使用Ctrl + C）
  2. 使用命令hexo d，同步到github。访问网站看看效果（使用github域名，一般都是博客仓库的名称）

删除博文

上面介绍了怎么写博文，如何发布博文，现在介绍一下如何删除博文，

  1. 先删掉source/_posts/目录下的文章.md（md后缀结尾的）
  2. 再删掉hexo根目录下面的db.json（记录数据库用的）
  3. 执行hexo cl && hexo g && hexo s命令，先在本地浏览器中预览一下，浏览器中输入http://localhost:4000预览一下本地效果。
  4. 如果没问题的话，则可以执行如下hexo d命令同步到github上
