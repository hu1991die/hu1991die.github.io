hexo系列教程：（三）将hexo博客绑定到自己的独立域名

  • Hexo
Blog

简介

通过前面两篇文章的学习，我们已经知道如何使用hexo+github Pages搭建一个属于自己的博客了，并且也大致地清楚了如何使用hexo发表文章了，但是此时我们的博客仍然是挂在github上面的，使用的是github pages提供的免费的域名空间，始终感觉还不是真正属于自己的，然后就一直想自己买一个域名，然后把这个博客绑定到自己的独立域名上，先前使用的还是github提供的域名，比如：hu1991die.github.io,现在想使用自己独有的域名，那么该怎么办呢？

域名购买与DNS的选择

  • Goddady
  • DNSPod

域名的购买，首选Goddady，原因如下：

  1. 在国外，不受国内各种干扰。
  2. 国外域名商中唯一支持支付宝付款的。
  3. 价格不高，还经常有优惠。
  4. 信誉好，口碑好，靠谱。
  5. 其实主要还是因为服务器比较稳定吧，其实github pages的空间也很稳定

goddady域名购买步骤：

  1. 首先打开goddady官网

  2. 在搜索框里面查询你想要购买的域名

  1. 查询到合适的域名之后选择“选择”按钮，之后点击“进入购物车”；

  1. 之后，默认不用管。之后是确认购买，修改购买权限，默认情况下是两年，可以修改成，看自己情况。另外，还可以在网上搜索goddady优惠码，会有不同程度的优惠，当然如果你是土豪，可以无视。

  1. 之后就是结算，和一般网购没啥区别，自行摸索即可

  2. 检查。结算后，重新登录，去「我的账户」，域名已经显示在你的账户了。（这里盗用一张图片）

  1. 补充一些注意事项：
  • 输入优惠码没有优惠或者优惠幅度较低，请清除浏览器 cookies 再尝试；
  • 如果没有支付宝支付选项，有可能是使用的优惠码不支持支付宝，请重新清除浏览器 cookies 再尝试；
  • 注册时用户填写信息时一定要输入正确的邮箱名字，否则修改十分麻烦。
  • 买完域名之后一定要记得去自己的邮箱查看激活邮件，否则域名激活不了。

当然，网上也有很多资料比较推荐万网，我暂时没用过，域名购买完成后就要在Goddady网站里设置dns服务商了。Goddady会默认提供两个国外的，对于国内网站和国内访问来说，最好使用国内的dns服务商，这里选择口碑最好使用最多的免费dns服务商dnspod.cn。

其中，Goddady是域名服务商，DNSPod是DNS服务商。

DNS设置

推荐使用 DNSpod，快，免费，稳定。

注册DNSpod，添加域名，如下图设置。

总共需要添加两条记录：一条@记录与一条www记录

说明：

1、其中，@记录的记录类型是A类型，指向的IP地址（也就是记录值）是github Pages的提供的ip，可以在cmd中ping一下你的github仓库的地址，比如ping hu1991die.github.io。之后将ping到的IP地址填写到记录值列中。

如博客不能登录，有可能是 github 更改了空间服务的 ip 地址，记得及时到在GitHub Pages查看最新的ip即可。

2、www记录的记录类型是CNAME类型，指向的IP地址（也就是记录值）是你在 github 注册的仓库地址（如：hu1991die.github.io）。

去 Godaddy 修改 DNS 地址

更改 godaddy 的 Nameservers 为 DNSpod 的 NameServers。

1、点击「My Account」，管理我的域名。

2、点击域名。

3、将 godaddy 的 Nameservers 更改成 f1g1ns1.dnspod.net 和 f1g1ns2.dnspod.net

如有不详看可以看DNSpod提供的官方帮助

到此为止，绑定工作就已经结束了，之后在浏览器中输入自己购买的独立域名，如果以上步骤配置正确的话是可以看到你的博客主页的，如果有问题就按照上面的步骤自己慢慢折腾吧！学习就是不停的折腾，哈哈！！
