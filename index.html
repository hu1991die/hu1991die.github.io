hexo系列教程：（二）hexo博客的目录结构与使用

了解Hexo

在上一篇文章中，我们只是使用Hexo简单地搭建了一个站点，但是要想更好地使用它，还是需要多了解关于Hexo的一些东西。

目录结构

Hexo的目录结构如下，其中：

  1. _config.yml是整个Hexo站点的核心配置文件

  2. source/_drafts是草稿文件目录

  3. source/_posts是发表的博文目录

  4. themes是下载的主题目录

至于其他的目录暂时可以不用管。

.
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
|   ├── _drafts
|   └── _posts
└── themes

首先就是修改_config.yml配置文件，主要就是配置一下自己的站点信息，配置文件中的注释比较详细，如果不懂的话可以看下官方文档，在预览状态下去改的话网页上能即时看到效果，这一点很赞，即时渲染。

我的如下：

title: 鬼脚飞's Blog                    #网站标题
subtitle: 没有梦想 何必远方！            #简介
description: 没有梦想 何必远方！            #描述
author: hu1991die                        #网站作者
language: zh-CN                            #网站语言
email: 347431996@qq.com                    #邮箱
timezone: Asia/Hong_Kong                #时区

默认的主题是landscape，更多的主题可以在https://github.com/hexojs/hexo/wiki/Themes这儿下载，基本就是git clone到themes目录下然后修改_config.yml中的theme字段即可，也十分简单方便。

例如：Yilia主题使用

安装主题

$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

配置主题

修改hexo根目录下的_config.yml ——》 theme: yilia

更新主题（随意，想更新就更新）

cd themes/yilia

git pull

Yilia主题效果展示

详情见博客主页：http://hu1991die.com/

Hexo常见指令

Hexo的常用命令有以下几个：

$ hexo new [layout] <title>     #建立新文章，默认在_posts下，layout="draft"时发布的是草稿
$ hexo publish <filename>       #将_drafts下的文件放到_posts下，也就是发布草稿
$ hexo generate                 #生成静态网页
$ hexo server                   #启动预览服务器，开启-d选项时可以预览草稿
$ hexo deploy                   #发布到远程服务器，开启--generate选项可以在deploy前自动generate

Tips

$ hexo n  == hexo new
$ hexo p  == hexo publish 
$ hexo g  == hexo generate
$ hexo s  == hexo server
$ hexo d  == hexo deploy

写博文

好了，站点配置好了，该了解的也都了解了，现在我想发表一篇博文，怎么做呢？

  1. 在hexo根目录下，执行命令

    hexo　new "my_first_post"

  2. 在H:\hexo\source\_posts中打开这个文件（打开方式用“记事本”或者NotePad++即可），配置开头头部信息，如下所示是本文的头部信息，以---结尾。接下来，使用Markdown编辑文章就可以了。

############—
title: hexo系列教程：（二）hexo博客的目录结构与使用
layout: post
date: 2016-03-07 16:37:43
comments: true
categories: Blog
tags: [Hexo]
keywords: Hexo, Blog
description: 生命在于折腾，又把博客折腾到Hexo了。给Hexo点赞。

############—

发布博文

由于生成的都是静态页面，可以和Octopress一样直接发布在gihub上。

  1. 使用命令hexo s，本地浏览器中访问http://localhost:4000预览效果。（退出server使用Ctrl + C）
  2. 使用命令hexo d，同步到github。访问网站看看效果（使用github域名，一般都是博客仓库的名称）

删除博文

上面介绍了怎么写博文，如何发布博文，现在介绍一下如何删除博文，

  1. 先删掉source/_posts/目录下的文章.md（md后缀结尾的）
  2. 再删掉hexo根目录下面的db.json（记录数据库用的）
  3. 执行hexo cl && hexo g && hexo s命令，先在本地浏览器中预览一下，浏览器中输入http://localhost:4000预览一下本地效果。
  4. 如果没问题的话，则可以执行如下hexo d命令同步到github上

好了，到了这里，我们已经学会了如何使用Hexo+Github Pages来搭建属于自己的博客，然后进行了一些基本配置，并且也学会了怎么发表博文。后面会陆续介绍一些高级点的个性化设置，比如使用多说的评论系统，绑定自己的独立域名等等，敬请期待哦！！！

hexo系列教程：（一）使用Hexo+GitHub Pages搭建个人博客

简介

什么 是hexo

hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github和Heroku上。作者是来自台湾的@tommy351。引用@tommy351的话，hexo：

快速、简单且功能强大的 Node.js 博客框架。
A fast, simple & powerful blog framework, powered by Node.js.

类似于jekyll、Octopress、Wordpress，我们可以用hexo创建自己的博客，托管到github或Heroku上，绑定自己的域名，用markdown写文章。本博客即使用hexo创建并托管在github上。先前在csdn上写过博客，虽然也是用的markdown语法（这里说明一下，markdown语法写博客真的很给力），后来一直想搭建一个真正属于自己的博客，网上搜了下wordpress,大致了解了一下，感觉挺麻烦的，后来又了解到了jekyll,也感觉挺繁琐，再到后来，了解到hexo，听说是jekyll的升级版，很方便。于是就花了几天时间研究了一下，确实很不错，效果也很赞！！！好了，说着说着就跑偏了，现在开始进入正题。

安装和配置

安装git（必须）

作用：用来把本地的hexo内容，比如写的文章提交到github仓库里面进行管理

安装：下载http://code.google.com/p/msysgit/并执行即可完成安装。如果用的是mac，安装XCode就自带会有git，这一块可以去网上搜一下，很多教程，也比较简单。

安装Node.js（必须）

作用：主要用来生成静态页面的
安装：到Node.js官网https://nodejs.org/en/下载相应平台的最新版本，一路安装即可。

申请Github账号（必须）

作用：是用来做博客的远程仓库、域名、服务器之类的。github仓库可以用来管理文章，如果自己没有域名的话（可以注册一个属于自己的域名），则可以使用github提供的免费域名，并且GithubPages默认会有300M的免费空间可供使用，资料自己管理，保存可靠，最主要的是稳定便利。所以将博客托管到github上进行管理再合适不过了。

正式安装Hexo

我们安装好git和Node.js之后，可以执行一下npm命令来安装hexo（注：Windows系统下可以在任意空白位置处鼠标右击，调出Git Bash命令窗口，Mac系统则直接打开终端Terminal即可）：

$sudo npm install -g hexo

初始化

然后，执行init命令初始化hexo到你指定的目录（比如：D:/hexo）,执行以下指令(在D:\hexo内空白处点击鼠标右键，选择Git bash)，hexo init即会自动在目标文件夹建立网站所需要的所有文件。或者：

$ sudo hexo init <folder> && cd <folder>

安装依赖包

$sudo npm install

好了，至此，全部的安装工作已经完成了！！

生成静态页面

首先，任意空白处右击调出git bash，cd到你的init目录（如：H:/dexo目录），执行如下命令，生成静态页面至hexo/public/目录中。

hexo generate (可简写 hexo g)

本地启动

启动本地服务，进行文章预览，命查看令：

hexo server （可简写hexo s）

然后再打开浏览器，在地址栏中输入http://localhost:4000进行预览

这里可能需要注意一下，有的伙伴可能输入这个地址之后啥也看不到，有可能是因为4000这个端口被某些服务给占用了，我当时也遇到过这种情况，原因是与福昕阅读器冲突了，后来杀掉这个福昕阅读器的进程之后就可以了，当然我们也可以更改hexo的端口，命令如下：

hexo server -p 5000 （可简写hexo s -p 5000）

然后到浏览器输入：http://localhost:5000再次预览查看。

注：如果一切正常的话，大伙肯定是会看到久违的Hello World主界面的。

ok，到这里，本地博客已经完全搭建起来了，但是，注意这里有个但是哦，这只是本地哦，现在的域名和服务器都是基于我们自己的电脑，其他的小伙伴是看不到你的这个Hello World主界面的。所以，我们需要部署到github上面去，与github进行关联。

关联GitHub

注册Github账号

如果先前已有账号的可以直接跳过，还没有的，点击这里https://github.com/signup/free进行注册，注册流程很简单，和一般的账号注册没啥区别，这里就不赘述了。

创建repository仓库

在自己的Github主页右下角，创建一个新的repository仓库。

比如我的Github账号是hu1991die,那么我创建的repository名字应该是hu1991die.github.io。注意，这个地方的命名需要注意一下，GitHubPage的名字必须用户名.github.com或者用户名.github.io。否则就变成了一个普通的项目了。

部署博客

编辑_config.xml（这个是hexo的核心配置文件，位于你的本地hexo博客目录下面，如H:\hexo下），在部署的时候，需要把下面的hu1991die替换成自己的github账号。

deploy:
      type: git
      repository: git@github.com:hu1991die/hu1991die.github.io.git
      branch: master

网上关于这个地方的type配置有些争议，不过据说官方最新版本的hexo中，这里的type要写成git，而不是github。

接下来，再执行下列命令即可完成github部署。

hexo clean（可简写hexo cl，清理）

hexo generate（可简写hexo g）

hexo deploy（可简写hexo d）

不过我一般都是这样的：

hexo cl && hexo g && hexo d && hexo cl

注意：有些新用户需要设置SSH，否则上述命令会失败。ssh 的介绍和设置方法请看官方教程，不用担心，很简单。就是将本地的SSH公钥添加到github上面，主要是做一个校验用。

记住：每次修改本地文件后，需要hexo generate才能保存。每次使用命令时，都要在D:\hexo目录下。

反正每次使用这行命令就行了。

hexo cl && hexo g && hexo d && hexo cl

好了,到了这里。我们的博客已经完全搭建起来了，打开浏览器，在浏览器输入地址中hu1991die.github.io就能看到你的成就了！小伙伴们，赶快去体验一下吧！
